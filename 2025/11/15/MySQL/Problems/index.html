MySQL 常见问题

发表于2025-11-15|更新于2025-11-15|技术栈
|总字数:1.5k|阅读时长:4分钟

Hash 索引与 BTree 索引有什么区别？

数据结构：Hash 基于哈希表，B-Tree 基于树结构。
适用场景：Hash 适合等值查询，B-Tree 相对适合范围查询。
性能：Hash 快速但不支持排序，B-Tree 支持排序。
存储空间：Hash 紧凑，B-Tree 详细。

怎么把一张表的部分数据更新到另一张表？

使用 UPDATE JOIN，UPFATE 确定要更新的表，再用 JOIN 子句将两张表连接起来，根据条件更新目标表的数据。

聚簇索引与非聚簇索引的区别？

聚簇索引与非聚簇索引的区别在于数据存储方式和访问效率。
聚簇索引将数据行存储在叶子节点，数据的物理顺序与索引顺序一致，适合范围查询和排序操作；
而非聚簇索引的叶子节点存储的是数据行的指针，数据的物理顺序与索引顺序无关，适合快速查找特定值。
聚簇索引通常只能有一个，而非聚簇索引可以有多个。

谈谈 SQL 注入及防范

SQL注入是指攻击者插入恶意 SQL 代码攻击数据库，产生数据泄露等风险。
防范措施包括：

  • 参数化查询（将 SQL 代码与输入分离，使得只能输入数字或字符串，而无法输入代码）
  • 输入验证
  • 最小权限原则
  • …

INNER / LEFT / RIGHT JOIN 区别？

INNER JOIN 只取交集，LEFT JOIN 保留左表全部，RIGHT JOIN 保留右表全部，未匹配部分用 NULL 填充，实际应用时根据业务需求选择合适的连接方式。

数据库为什么使用B+树而不使用红黑树？

B+树更适合磁盘存储和范围查询，B+数包含多个键值，减少了树的高度，从而减少了磁盘IO的次数，提高查询效率。B+树的叶子节点通过链表链接，支持于顺序查询和范围查询。红黑树是二叉树，且节点大小固定，存储大量数据时，树高增长过快，查找性能下降。

大表（千万级）查询 / 维护该怎么办？

索引优化，拒绝全表扫描，语句调优 避免 SELECT *，明确指定所需字段，减少I/O开销，分区，分页，分表

如何高效批量插入数据？

  • 使用批量插入语句，一次插入多行
  • 插入前暂时禁用索引，插入后再重建索引
  • 多个插入操作放在一个事务，减少事务提交的开销

DELETE / TRUNCATE / DROP 区别？

delete 删除数据行，drop 删除表数据和表结构，truncate 清空表的所有数据

LIMIT 100000000,10 与 LIMIT 10 性能差异？

LIMIT 100000000, 10 等价于 LIMIT 10 OFFSET 100000000, 都用于跳过指定的 “偏移量” 行, 返回指定条数的行
即使只取 10 行，也需从头遍历到第 1 亿行，过程中会读取大量无关数据，查询速度极慢。
OFFSET 仅适合小偏移量（如前几页）分页，大数据量分页通常需用“游标”(基于主键/唯一索引的范围查询) 替代，如 WHERE id > 100000000 LIMIT 10

WHERE 与 HAVING 的区别？

where 分组前过滤不可以使用聚合函数，having 分组后过滤可以使用聚合函数

联合索引的存储结构与最左前缀原则

联合索引的存储结构是将多个列的值组合在一起形成一个复合键，按照字典序进行排序存储。
最左前缀原则指的是在使用联合索引时，查询条件必须从索引的最左列开始，才能有效利用索引进行加速查询。

如何使用 EXPLAIN 分析查询？关注哪些列？

使用 EXPLAIN 分析查询只需要在SELECT语句前加上“EXPLAIN”即可。
关注的列有：

  1. type访问类型；
  2. rows预估需要扫描的行数；
  3. keys实际使用的索引；
  4. extra额外信息；
  5. possible_keys可能使用的索引；
  6. id执行顺序；
  7. select_type查询类型；
  8. tables查询涉及的表等。

数据库分页的实现方式？

一般是通过 LIMIT 和 OFFSET 语句结合实现分页，LIMIT返回记录的数量，OFFSET指定记录的起始位置，但是数据量较大时，可以使用基于索引的分页方式，就是记录上次查询的最后一个主键值来获取下一页的索引，从而避免使用 OFFSET，不需要读取并跳过之前的数据，提高查询速度。

B+ 树查询数据的全过程？

在B+树中，由叶子节点存储实际的数据行记录，查询数据是层级递归与页内二分+链表遍历的结合
查找过程：

查询从B+树的根节点开始，通过比较目标键值与节点中存储的索引键值（通常采用二分查找），确定数据落在哪个区间，从而选择对应的分支继续向下查找。这一过程逐层递归，从上到下最终定位到目标叶子节点。
叶子节点大小为16KB，存储的数据行不止一条，对叶子节点中数据行以组的形式划分，利用页目录结构，通过二分查找页目录中的槽，可以快速定位到包含目标数据行的组。
定位到目标组后，通过链表遍历（通常是单向或双向链表）逐条检查数据行，直到找到完全匹配的记录。

索引失效后怎么办？

  1. 优化sql结构，满足最左前缀原则；
  2. 尽量减少索引列的数学计算；
  3. 字段类型尽量符合定义，减少隐式转换
  4. 根据需要优化索引结构，提高索引的选择性

EXISTS 与 IN 的区别？

EXISTS 遍历外表每一行，检查下内表有没有存在，适合外表小内表大的场景

IN先执行内表查询，再把结果集进行匹配，适合内表小外表大的情景
